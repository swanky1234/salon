## 🔵 Continue with Google — Setup Steps

The code is already written. You just need to enable it in **Supabase**:

**Step 1 – Go to your Supabase project**
> https://app.supabase.com → Your Project → Authentication → Providers

**Step 2 – Enable Google Provider**
1. Find **Google** in the list and toggle it **ON**
2. You'll need a **Client ID** and **Client Secret** from Google Cloud Console

**Step 3 – Create Google OAuth credentials**
1. Go to https://console.cloud.google.com
2. Create a new project (or use existing)
3. Go to **APIs & Services → Credentials**
4. Click **Create Credentials → OAuth 2.0 Client ID**
5. Application type: **Web application**
6. Add Authorized redirect URI:
   ```
   https://vppntsrbpdnmdwkllfal.supabase.co/auth/v1/callback
   ```
7. Copy the **Client ID** and **Client Secret**

**Step 4 – Paste into Supabase**
- Paste Client ID and Client Secret into the Google provider settings in Supabase
- Save

**Step 5 – Add your site URL to Supabase**
> Authentication → URL Configuration
- **Site URL**: `http://localhost:5173` (for dev), your domain for production
- **Redirect URLs**: Add `http://localhost:5173/`

That's it — the "Continue with Google" button will work immediately.

## 🗄️ Fix Supabase Services Not Loading from DB

Run **`supabase_rls_fix.sql`** in your Supabase SQL Editor:
> https://app.supabase.com → SQL Editor → Paste file contents → Run

This will:
- Enable public **SELECT** on `services`, `stylists`, `holidays`, `availability`, `settings`
- Enable authenticated access to `bookings` and `favorites`
- **Seed the 18 default services** if the table is empty

> [!NOTE]
> Even without running this SQL, the app works using the **static fallback** — all 18 services appear on the website. The SQL just ensures the admin panel's services sync to the public site.

## 🖼️ Image Upload — Supabase Storage Buckets Required

For image uploads to work, ensure these **storage buckets** exist in Supabase:
> Storage → Create new bucket

| Bucket name | Public |
|---|---|
| `service-images` | ✅ Yes |
| `stylist-images` | ✅ Yes |

Both need to be **public** so uploaded images display correctly.
